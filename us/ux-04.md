# US-UX-04: Sistema de notificaciones Toast

> **Epic:** UX · **Prioridad:** P1 · **Estado impl:** ❌ No existe  
> **Dependencias:** Ninguna (transversal)  
> **Bloquea:** Todas las US que necesiten feedback de acción

## Historia

**Como** jugador  
**quiero** ver confirmaciones visuales cuando realizo acciones (agregar juego, cambiar estado, etc.)  
**para** saber que mi acción se completó sin tener que buscar el cambio en la pantalla.

---

## Criterios de Aceptación

### AC-1: Toast en acciones exitosas
```gherkin
Dado que agrego un juego a la biblioteca desde SearchView
Cuando la acción se completa
Entonces veo un toast en la esquina inferior derecha:
  "Hollow Knight agregado a tu Biblioteca"
  Y el toast desaparece automáticamente después de 4 segundos
```

### AC-2: Toast en errores
```gherkin
Dado que intento una acción que falla
Cuando el backend devuelve un error
Entonces veo un toast rojo con el mensaje de error
  Y el toast se queda hasta que el usuario lo cierre (no auto-dismiss en errores)
```

### AC-3: Toast no bloqueante
```gherkin
Dado que hay un toast visible
Cuando interactúo con el resto de la app
Entonces el toast no bloquea la UI
  Y puedo cerrarlo con X o swipe
```

### AC-4: Stacking de toasts
```gherkin
Dado que hago múltiples acciones rápidas
Cuando se acumulan toasts
Entonces se apilan verticalmente (máximo 3 visibles)
  Y los más viejos se descartan
```

## Scenarios

| # | Tipo | Escenario | Resultado esperado |
|---|------|-----------|-------------------|
| 1 | Happy | Agregar juego exitoso | Toast verde/neutro "Juego agregado" |
| 2 | Happy | Cambiar estado exitoso | Toast "Estado actualizado" |
| 3 | Happy | Eliminar juego | Toast "Juego eliminado" |
| 4 | Sad | Error de red | Toast rojo persistente con mensaje |
| 5 | Sad | Duplicado (409) | Toast ámbar "Ya tenés este juego" |
| 6 | Edge | 5 acciones rápidas | Solo 3 toasts visibles, stack |
| 7 | Edge | Toast + ConfirmDialog abierto | Ambos visibles sin conflicto de z-index |
| 8 | Border | Mensaje largo (error detallado) | Toast expandido o truncado con "Ver más" |

---

## Acciones que necesitan toast (inventario)
- Agregar juego a biblioteca (SearchView → backlog)
- Agregar juego a wishlist (SearchView → wishlist)
- Cambiar estado de juego (Dashboard)
- Cambiar plataforma de juego (Dashboard)
- Eliminar juego (Dashboard, post-confirm)
- Crear watch de precio (WishlistView)
- Marcar alerta como leída (AlertsView)
- Guardar onboarding (UserOnboarding)
- Error genérico de API (client.js interceptor)
