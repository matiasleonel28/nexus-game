# US-UX-07: Migrar colores hardcodeados a design tokens

## Historia
Como jugador, quiero que la interfaz use colores consistentes definidos por design tokens para que el tema visual sea coherente en toda la app.

## Criterios de Aceptacion

### AC-1: Reemplazar grises de texto por tokens
```gherkin
Dado que varios componentes usan text-gray-400, text-gray-500 o text-gray-600
Cuando se migran a text-[var(--muted)]
Entonces el color del texto secundario es consistente en toda la app
Y responde al theme activo
```

### AC-2: Reemplazar bordes hardcodeados por token --line
```gherkin
Dado que hay componentes con border-gray-700 o border-gray-800
Cuando se migran a border-[var(--line)]
Entonces los bordes usan el token de linea del sistema de diseno
```

### AC-3: Reemplazar texto principal hardcodeado
```gherkin
Dado que algunos componentes usan text-gray-200 o text-gray-300 para texto principal
Cuando se migran a text-[var(--text)]
Entonces el texto principal usa el token correspondiente
```

### AC-4: Reemplazar accent y danger hardcodeados
```gherkin
Dado que hay componentes con hover:bg-amber-400 o border-red-900
Cuando se migran a hover:bg-[var(--accent)] y border-[var(--danger)]
Entonces los colores de accion y error usan tokens del sistema
```

### AC-5: NotFound.jsx queda excluido de esta US
```gherkin
Dado que NotFound.jsx usa una paleta indigo completamente distinta
Cuando se completa esta US
Entonces NotFound.jsx no se modifica (se cubre en UX-08)
```

## Scenarios

| # | Tipo | Escenario | Input | Expected |
|---|------|-----------|-------|----------|
| 1 | Happy | Texto muted consistente | Abrir HunterView y WishlistView | Ambos usan el mismo color --muted para texto secundario |
| 2 | Happy | Bordes unificados | Abrir Dashboard y AlertsView | Bordes usan --line, visualmente identicos |
| 3 | Happy | Hover accent funciona | Hover sobre boton en SearchView | Color de hover es var(--accent) |
| 4 | Happy | Danger border funciona | Alerta de error en AlertsView | Borde usa var(--danger) |
| 5 | Sad | Token CSS no definido | Variable --muted no existe en :root | El componente queda sin color visible (transparente) |
| 6 | Sad | Clase Tailwind invalida | Typo en var(--muuted) | Build falla o color no se aplica |
| 7 | Happy | grep no encuentra hardcoded | grep text-gray en src/ | 0 resultados (excepto NotFound.jsx) |

## Edge / Border Cases
- Algunos componentes podrian usar `className` dinamico con template literals; verificar que el reemplazo funcione con interpolacion.
- Si algun color gray se usa para disabled states, mapear a --muted con opacity en vez de --text.
- ConfirmDialog.jsx puede tener variantes (danger vs info); asegurarse de que ambas usen tokens.
- Verificar que los tokens estan definidos en el CSS global antes de migrar (si no, definirlos primero).
