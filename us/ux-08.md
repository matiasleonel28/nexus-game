# US-UX-08: Redisenar NotFound.jsx con design tokens

## Historia
Como jugador, quiero que la pagina 404 tenga el mismo estilo visual que el resto de Nexus para que la experiencia sea coherente aunque me pierda.

## Criterios de Aceptacion

### AC-1: Reemplazar paleta indigo/zinc por tokens del proyecto
```gherkin
Dado que NotFound.jsx usa zinc-950, indigo-600, zinc-400, zinc-500
Cuando se redisena el componente
Entonces usa los tokens --ink, --surface, --accent, --muted y --line
Y no queda ningun color hardcodeado de Tailwind
```

### AC-2: Reemplazar botones rounded-full por estilo del proyecto
```gherkin
Dado que NotFound.jsx tiene botones con rounded-full y estilos propios
Cuando se redisena
Entonces los botones siguen el patron de botones del resto de la app
```

### AC-3: Mantener contenido en espanol
```gherkin
Dado que el mensaje de 404 puede estar en ingles
Cuando se redisena el componente
Entonces el titulo, subtitulo y texto del boton estan en espanol
Y el tono es neutro y claro
```

### AC-4: Fondo coherente con el layout general
```gherkin
Dado que NotFound.jsx usa bg-zinc-950 como fondo propio
Cuando se redisena
Entonces usa bg-[var(--ink)] o hereda el fondo del layout
Y no hay flash de color distinto al navegar a una ruta inexistente
```

## Scenarios

| # | Tipo | Escenario | Input | Expected |
|---|------|-----------|-------|----------|
| 1 | Happy | Pagina 404 con estilo Nexus | Navegar a /ruta-inexistente | Se muestra 404 con tokens del proyecto, visualmente integrada |
| 2 | Happy | Boton volver funciona | Click en boton de la 404 | Navega al home o a la pagina anterior |
| 3 | Happy | Responsive | 404 en mobile (375px) | Layout se adapta sin overflow horizontal |
| 4 | Sad | Tokens no cargados | CSS vars no definidas | La pagina sigue siendo legible (fallback razonable) |
| 5 | Happy | Sin colores hardcodeados | grep indigo NotFound.jsx | 0 resultados |
| 6 | Happy | Sin colores zinc | grep zinc NotFound.jsx | 0 resultados |

## Edge / Border Cases
- Si el layout ya provee el fondo --ink, NotFound no deberia redefinirlo (evitar doble background).
- Verificar que el componente no importe estilos de un CSS module propio con colores hardcodeados.
- Si hay animaciones (transitions en hover), mantenerlas pero usando tokens.
- El boton de "volver" debe funcionar tanto como link a "/" como con navigate(-1); definir cual corresponde.
